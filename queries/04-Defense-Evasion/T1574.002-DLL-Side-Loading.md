# Suspicious DLL Side-Loading from User-Writable Directories

**MITRE Technique:** T1574.002 - Hijack Execution Flow: DLL Side-Loading  
**MITRE Tactic:** TA0005 - Defense Evasion  
**Severity:** High  
**False Positive Rate:** High  

## Description
Detects DLL files being loaded from user-writable directories such as AppData, Temp, and Downloads. Attackers plant malicious DLLs in these locations to hijack legitimate application execution. High false positive rate because many legitimate applications such as Zoom, Slack, and Teams install and load DLLs from AppData.

## KQL Query
```kql
DeviceImageLoadEvents
| where Timestamp > ago(24h)
| where FolderPath has_any (
    "\\AppData\\",
    "\\Temp\\",
    "\\Downloads\\",
    "\\Public\\"
)
| where FileName endswith ".dll"
| where InitiatingProcessFileName !in~ (
    "MsMpEng.exe",
    "senseir.exe",
    "installer.exe",
    "msiexec.exe"
)
| project
    Timestamp,
    DeviceName,
    FileName,
    FolderPath,
    InitiatingProcessFileName,
    InitiatingProcessAccountName,
    SHA256
| order by Timestamp desc
```

## Results in Lab
- **Hit found:** YES — 35 results
- **Initiating process:** zoom.exe
- **Location:** C:\Users\HP\AppData\Roaming\...
- **Account:** hp
- **Verdict:** FALSE POSITIVE — Zoom legitimately installs and loads all its DLLs from AppData because it does not require administrator rights to install

## False Positive Analysis
This is a high false positive query requiring significant environment-specific tuning. Investigation workflow:
1. Identify initiating process — zoom.exe (known legitimate application)
2. Identify DLL names — all prefixed with "z" indicating Zoom's own DLLs
3. Verify SHA256 hashes against known-good Zoom installer hashes
4. Conclusion — legitimate Zoom DLL loading, not malicious side-loading

## Tuned Query (Production-Ready)
```kql
DeviceImageLoadEvents
| where Timestamp > ago(24h)
| where FolderPath has_any (
    "\\AppData\\",
    "\\Temp\\",
    "\\Downloads\\",
    "\\Public\\"
)
| where FileName endswith ".dll"
| where InitiatingProcessFileName !in~ (
    "MsMpEng.exe",
    "senseir.exe",
    "installer.exe",
    "msiexec.exe",
    "zoom.exe",
    "slack.exe",
    "teams.exe",
    "discord.exe"
)
| project
    Timestamp,
    DeviceName,
    FileName,
    FolderPath,
    InitiatingProcessFileName,
    InitiatingProcessAccountName,
    SHA256
| order by Timestamp desc
```

## Tuning Notes
- Whitelist known user-installed applications by InitiatingProcessFileName
- Focus on DLLs loaded by unexpected processes — browsers, Office apps, system processes
- Submit unknown SHA256 hashes to VirusTotal for reputation checking
