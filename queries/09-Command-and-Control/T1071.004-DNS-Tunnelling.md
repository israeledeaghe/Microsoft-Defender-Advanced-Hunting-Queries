# DNS Tunnelling Detection

**MITRE Technique:** T1071.004 - Application Layer Protocol: DNS  
**MITRE Tactic:** TA0011 - Command and Control  
**Severity:** High  
**False Positive Rate:** Medium  

## Description
Detects abnormally long DNS queries which may indicate DNS tunnelling. Attackers use DNS tunnelling to exfiltrate data or establish covert C2 channels by encoding data within DNS query strings. Legitimate DNS queries are typically short — queries over 50 characters warrant investigation.

## KQL Query
```kql
DeviceNetworkEvents
| where Timestamp > ago(24h)
| where ActionType == "DnsConnectionInspected"
| extend QueryLength = strlen(RemoteUrl)
| where QueryLength > 50
| where RemoteUrl !endswith ".microsoft.com"
    and RemoteUrl !endswith ".windows.com"
    and RemoteUrl !endswith ".windowsupdate.com"
    and RemoteUrl !endswith ".office.com"
| project
    Timestamp,
    DeviceName,
    RemoteUrl,
    QueryLength,
    InitiatingProcessFileName,
    InitiatingProcessAccountName,
    RemoteIP
| order by QueryLength desc
```

## Results in Lab
- **Hit found:** No results
- **Verdict:** Expected baseline — no abnormally long DNS queries observed
- **Notes:** Query validated against live MDE P2 telemetry on Windows 11 Pro

## Tuning Notes
- Adjust query length threshold based on your environment's baseline DNS behaviour
- Add additional trusted domains to the exclusion list
- Correlate RemoteIP with threat intelligence feeds for enrichment
