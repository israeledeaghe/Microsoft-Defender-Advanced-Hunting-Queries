# Local Data Staging for Exfiltration

**MITRE Technique:** T1074.001 - Local Data Staging  
**MITRE Tactic:** TA0009 - Collection  
**Severity:** High  
**False Positive Rate:** Medium  

## Description
Detects archive files being created in temporary or public directories, indicating data may be staged for exfiltration. Attackers commonly compress sensitive files into archives before transferring them out of the network to reduce transfer size and evade data loss prevention tools.

## KQL Query
```kql
DeviceFileEvents
| where Timestamp > ago(24h)
| where ActionType in ("FileCreated", "FileCopied")
| where FolderPath has_any (
    "\\Temp\\",
    "\\AppData\\Local\\Temp\\",
    "\\Public\\",
    "\\ProgramData\\"
)
| where FileName endswith ".zip"
    or FileName endswith ".rar"
    or FileName endswith ".7z"
    or FileName endswith ".tar"
    or FileName endswith ".gz"
| project
    Timestamp,
    DeviceName,
    FileName,
    FolderPath,
    InitiatingProcessFileName,
    InitiatingProcessAccountName,
    SHA256
| order by Timestamp desc
```

## Results in Lab
- **Hit found:** No results
- **Verdict:** Expected baseline — no archive creation in staging directories observed
- **Notes:** Query validated against live MDE P2 telemetry on Windows 11 Pro

## Tuning Notes
- Exclude known backup software by whitelisting InitiatingProcessFileName
- Correlate with DeviceNetworkEvents to check for subsequent large outbound transfers
- Flag archives created shortly before large network uploads as highest priority
