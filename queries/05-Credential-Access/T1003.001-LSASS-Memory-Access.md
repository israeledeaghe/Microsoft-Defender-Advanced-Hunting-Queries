# LSASS Memory Access (Credential Dumping)

**MITRE Technique:** T1003.001 - OS Credential Dumping: LSASS Memory  
**MITRE Tactic:** TA0006 - Credential Access  
**Severity:** Critical  
**False Positive Rate:** Low  

## Description
Detects processes attempting to access LSASS memory. Tools like Mimikatz, ProcDump, and Task Manager can dump LSASS memory to extract plaintext passwords, NTLM hashes, and Kerberos tickets. Any unexpected process touching LSASS should be treated as critical.

## KQL Query
```kql
DeviceProcessEvents
| where Timestamp > ago(24h)
| where FileName =~ "lsass.exe" 
    or ProcessCommandLine has "lsass"
    or InitiatingProcessCommandLine has "lsass"
| where InitiatingProcessFileName !in~ (
    "MsMpEng.exe",
    "senseir.exe",
    "MsSense.exe",
    "csrss.exe",
    "wininit.exe"
)
| project
    Timestamp,
    DeviceName,
    FileName,
    ProcessCommandLine,
    InitiatingProcessFileName,
    InitiatingProcessAccountName,
    InitiatingProcessCommandLine
| order by Timestamp desc
```

## Results in Lab
- **Hit found:** No results
- **Verdict:** Expected baseline — no LSASS access observed outside trusted processes
- **Notes:** Query validated against live MDE P2 telemetry on Windows 11 Pro

## Tuning Notes
- Any hit from an unexpected process is critical — investigate immediately
- Exclude security tools like MsMpEng.exe (Windows Defender) and senseir.exe (MDE sensor)
- Correlate with DeviceLogonEvents to check for subsequent lateral movement
