# PowerShell Download Cradle Detection

**MITRE Technique:** T1059.001 - Command and Scripting Interpreter: PowerShell  
**MITRE Tactic:** TA0002 - Execution  
**Severity:** Critical  
**False Positive Rate:** Low  

## Description
Detects PowerShell being used as a download cradle to fetch and execute payloads directly in memory. This technique bypasses traditional file-based antivirus detection because the payload never touches disk. Commonly used in fileless malware attacks, ransomware delivery, and post-exploitation frameworks like Cobalt Strike and Metasploit.

## KQL Query
```kql
DeviceProcessEvents
| where Timestamp > ago(24h)
| where FileName in~ ("powershell.exe", "pwsh.exe")
| where ProcessCommandLine has_any (
    "Net.WebClient",
    "WebRequest",
    "DownloadFile",
    "DownloadString",
    "DownloadData",
    "OpenRead",
    "WebProxy",
    "Start-BitsTransfer",
    "Invoke-WebRequest",
    "curl ",
    "wget "
)
| project
    Timestamp,
    DeviceName,
    AccountName,
    FileName,
    ProcessCommandLine,
    InitiatingProcessFileName
| order by Timestamp desc
```

## Results in Lab
- **Hit found:** No results
- **Verdict:** Expected baseline — no PowerShell download cradles observed
- **Notes:** Query validated against live MDE P2 telemetry on Windows 11 Pro

## Tuning Notes
- Very low false positive rate in most environments
- Legitimate use of Invoke-WebRequest in admin scripts should be whitelisted by AccountName
- Correlate with DeviceNetworkEvents to identify the download destination IP
