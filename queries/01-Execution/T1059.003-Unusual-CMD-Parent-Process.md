# Unusual Parent Process Spawning CMD

**MITRE Technique:** T1059.003 - Windows Command Shell  
**MITRE Tactic:** TA0002 - Execution  
**Severity:** High  
**False Positive Rate:** Medium  

## Description
Detects cmd.exe being spawned by unusual parent processes. While CMD is a legitimate Windows tool, attackers abuse it for payload execution, reconnaissance, and lateral movement. Unexpected parent processes spawning CMD — such as database engines, web servers, or document readers — are strong indicators of compromise.

## KQL Query
```kql
DeviceProcessEvents
| where Timestamp > ago(24h)
| where FileName =~ "cmd.exe"
| where InitiatingProcessFileName !in~ (
    "explorer.exe",
    "cmd.exe",
    "powershell.exe",
    "pwsh.exe",
    "svchost.exe",
    "services.exe",
    "MsMpEng.exe",
    "senseir.exe",
    "conhost.exe",
    "devenv.exe",
    "code.exe"
)
| project
    Timestamp,
    DeviceName,
    FileName,
    ProcessCommandLine,
    InitiatingProcessFileName,
    InitiatingProcessCommandLine,
    InitiatingProcessAccountName
| order by Timestamp desc
```

## Results in Lab
- **Hit found:** YES — 38 results
- **Initiating processes:** postgres.exe, python3.13.exe
- **Account:** splunkd
- **Verdict:** FALSE POSITIVE — Splunk is installed on this machine and routinely spawns cmd.exe via its internal PostgreSQL database and Python processes as part of normal operation

## False Positive Analysis
This is a textbook example of false positive analysis. In a production SOC environment, the investigation workflow would be:
1. Identify the initiating process — postgres.exe and python3.13.exe
2. Identify the account — splunkd (a known service account)
3. Review the command line — cmd.exe /c ver and postgres administration commands
4. Conclusion — legitimate Splunk internal activity, not malicious

## Tuned Query (Production-Ready)
```kql
DeviceProcessEvents
| where Timestamp > ago(24h)
| where FileName =~ "cmd.exe"
| where InitiatingProcessFileName !in~ (
    "explorer.exe",
    "cmd.exe",
    "powershell.exe",
    "pwsh.exe",
    "svchost.exe",
    "services.exe",
    "MsMpEng.exe",
    "senseir.exe",
    "conhost.exe",
    "devenv.exe",
    "code.exe",
    "postgres.exe",
    "python3.13.exe"
)
| where InitiatingProcessAccountName !in~ ("splunkd", "system")
| project
    Timestamp,
    DeviceName,
    FileName,
    ProcessCommandLine,
    InitiatingProcessFileName,
    InitiatingProcessCommandLine,
    InitiatingProcessAccountName
| order by Timestamp desc
```

## Tuning Notes
- Always investigate the initiating process and account before dismissing hits
- Add legitimate software service accounts to the exclusion list
- This query demonstrates the importance of environment-specific tuning
