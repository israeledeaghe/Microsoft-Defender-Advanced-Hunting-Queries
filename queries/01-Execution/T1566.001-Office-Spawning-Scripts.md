# Suspicious Script Execution from Office Applications

**MITRE Technique:** T1566.001 - Phishing: Spearphishing Attachment  
**MITRE Tactic:** TA0001 - Initial Access  
**Severity:** Critical  
**False Positive Rate:** Low  

## Description
Detects Office applications spawning script interpreters or command-line tools. This is the classic indicator of malicious macro execution from phishing attachments. Legitimate Office applications should never directly spawn PowerShell, CMD, or script hosts.

## KQL Query
```kql
DeviceProcessEvents
| where Timestamp > ago(24h)
| where InitiatingProcessFileName in~ (
    "winword.exe",
    "excel.exe",
    "powerpnt.exe",
    "outlook.exe",
    "onenote.exe"
)
| where FileName in~ (
    "powershell.exe",
    "cmd.exe",
    "wscript.exe",
    "cscript.exe",
    "mshta.exe",
    "rundll32.exe",
    "regsvr32.exe",
    "certutil.exe"
)
| project
    Timestamp,
    DeviceName,
    InitiatingProcessFileName,
    FileName,
    ProcessCommandLine,
    InitiatingProcessCommandLine,
    InitiatingProcessAccountName
| order by Timestamp desc
```

## Results in Lab
- **Hit found:** No results
- **Verdict:** Expected baseline — no Office applications spawning script interpreters observed
- **Notes:** Query validated against live MDE P2 telemetry on Windows 11 Pro

## Tuning Notes
- Any hit on this query should be treated as critical — no exceptions
- Extend the initiating process list to include any document viewer in your environment
- Correlate with email logs to identify the source phishing email
