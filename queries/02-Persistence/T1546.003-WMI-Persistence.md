# WMI Persistence via Event Subscription

**MITRE Technique:** T1546.003 - Windows Management Instrumentation Event Subscription  
**MITRE Tactic:** TA0003 - Persistence  
**Severity:** Critical  
**False Positive Rate:** Low  

## Description
Detects WMI being used to establish persistence through event subscriptions. WMI subscriptions execute payloads in response to system events (e.g. user logon, process creation) and survive reboots. This technique is difficult to detect because WMI activity blends into normal Windows operation.

## KQL Query
```kql
DeviceProcessEvents
| where Timestamp > ago(24h)
| where FileName =~ "wmic.exe" or FileName =~ "wmiprvse.exe"
| where ProcessCommandLine has_any (
    "process call create",
    "ActiveScriptEventConsumer",
    "CommandLineEventConsumer",
    "EventFilter",
    "EventConsumer",
    "FilterToConsumerBinding"
)
| project
    Timestamp,
    DeviceName,
    FileName,
    ProcessCommandLine,
    InitiatingProcessFileName,
    InitiatingProcessAccountName
| order by Timestamp desc
```

## Results in Lab
- **Hit found:** No results
- **Verdict:** Expected baseline — no WMI persistence observed
- **Notes:** Query validated against live MDE P2 telemetry on Windows 11 Pro

## Tuning Notes
- Any hit on this query should be treated as critical and investigated immediately
- WMI persistence is rarely used by legitimate software
- Correlate with DeviceRegistryEvents to check for associated MOF file drops
