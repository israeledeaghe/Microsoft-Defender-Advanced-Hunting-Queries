# Suspicious Local Account Creation

**MITRE Technique:** T1136.001 - Create Account: Local Account  
**MITRE Tactic:** TA0003 - Persistence  
**Severity:** High  
**False Positive Rate:** Low  

## Description
Detects new local user accounts being created or added to the Administrators group via net.exe. Attackers create backdoor accounts to maintain persistent access even after credential resets. Adding accounts to the local Administrators group immediately escalates their privileges.

## KQL Query
```kql
DeviceProcessEvents
| where Timestamp > ago(24h)
| where FileName in~ ("net.exe", "net1.exe")
| where ProcessCommandLine has_any (
    "user /add",
    "localgroup administrators",
    "user add"
)
| project
    Timestamp,
    DeviceName,
    FileName,
    ProcessCommandLine,
    InitiatingProcessFileName,
    InitiatingProcessAccountName
| order by Timestamp desc
```

## Results in Lab
- **Hit found:** No results
- **Verdict:** Expected baseline — no suspicious account creation observed
- **Notes:** Query validated against live MDE P2 telemetry on Windows 11 Pro

## Tuning Notes
- Any hit outside of known IT provisioning windows should be investigated immediately
- Whitelist known IT admin accounts by InitiatingProcessAccountName
- Correlate with DeviceLogonEvents to track subsequent logons from the new account
