# Registry Run Key Persistence

**MITRE Technique:** T1547.001 - Registry Run Keys / Startup Folder  
**MITRE Tactic:** TA0003 - Persistence  
**Severity:** High  
**False Positive Rate:** Medium  

## Description
Detects modifications to registry autorun keys commonly abused by attackers to maintain persistence across reboots. Writing to Run/RunOnce keys causes the specified payload to execute automatically every time a user logs in or the system starts.

## KQL Query
```kql
DeviceRegistryEvents
| where Timestamp > ago(24h)
| where ActionType in ("RegistryValueSet", "RegistryKeyCreated")
| where RegistryKey has_any (
    "\\Software\\Microsoft\\Windows\\CurrentVersion\\Run",
    "\\Software\\Microsoft\\Windows\\CurrentVersion\\RunOnce",
    "\\Software\\Wow6432Node\\Microsoft\\Windows\\CurrentVersion\\Run",
    "\\SYSTEM\\CurrentControlSet\\Services"
)
| project
    Timestamp,
    DeviceName,
    ActionType,
    RegistryKey,
    RegistryValueName,
    RegistryValueData,
    InitiatingProcessFileName,
    InitiatingProcessAccountName,
    InitiatingProcessCommandLine
| order by Timestamp desc
```

## Results in Lab
- **Hit found:** No results
- **Verdict:** Expected baseline — no registry autorun modifications observed in last 24 hours
- **Notes:** Query validated against live MDE P2 telemetry on Windows 11 Pro

## Tuning Notes
- Exclude known software installers by whitelisting InitiatingProcessFileName
- Any hit from an unexpected process should be treated as high priority
- Correlate RegistryValueData with DeviceFileEvents to confirm the referenced executable exists
